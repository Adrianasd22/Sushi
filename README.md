

## Convención de commits:
Casi siempre se usa **feat(scoope):description**
- `feat:` añade funcionalidad, un componente, imagen ...
- `refactor:` reorganizar codigo
- `style:` cambios visuales o modificar algo.
- `docs:` documentacion o cambio de nombre / correccion ortográfica.
- `fix:` arreglar un error.
- `test:` tests
- `chore:` cambio en dependencias
- `build:` cambios en el build
- `BREAKING CHANGE:` cambio grande (cambio de version --> MAYOR)

Ej: 
```
feat(users): add user profile component
feat(home): add hero banner image
feat(routing): add route for admin dashboard
refactor(routing): reorganize route structure
chore: update Angular to v17
style(layout): adjust navbar spacing
style(home): change background image
fix(header): correct logo path
fix(products): fix missing product image
```
